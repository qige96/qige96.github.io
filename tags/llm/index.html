AI Agent 的工作流设计模式

最近在做 AI Agent 相关的项目，梳理了几种常见的工作流设计模式。记录一下，也作为后续迭代的参考。

1. 单 Agent + 工具调用（ReAct 模式）

最简单的模式。一个 LLM 实例，通过 function calling 使用外部工具，循环执行直到任务完成。

# 伪代码
agent = Agent(tools=[search, calculator, code_exec])

while not done:
    thought = agent.think(task)
    action = agent.act(thought)
    observation = tool.execute(action)
    agent.observe(observation)

适用场景: 任务边界清晰、工具明确的场景，如数据分析、信息检索。

缺点: 复杂任务容易陷入循环，上下文膨胀快。

2. 多 Agent 协作（Multi-Agent）

多个 Agent 分工协作，通过消息传递或共享状态完成任务。

常见的分工方式：

角色职责
Planner拆解任务、制定计划
Executor执行具体子任务
Critic检查结果、提出改进
Memory管理长期记忆、知识检索

实现方式:

  • 层级式: 一个 Orchestrator 调度多个 Worker Agent
  • 对等式: Agent 之间直接通信（类似微服务）
  • 流水线式: 固定步骤链，每个步骤一个 Agent

适用场景: 复杂多步骤任务，如软件开发、研究报告生成。
